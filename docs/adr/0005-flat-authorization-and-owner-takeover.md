# Flat authorization: every authenticated user is an equal triager, ownership is assignable to anyone

There are no roles and no per-Request permissions. Any authenticated user can change any Request's status and priority, add notes, and set or clear ownership — everyone is an equal triager. This is an explicit non-goal, not an omission: RBAC, teams, and admin are deliberately out of scope for a small internal triage tool, and adding them would touch every Server Action and most of the UI. Ownership can be assigned to *any* authenticated user, not only oneself: the picker lists every user in the Clerk instance (a "Me" shortcut is pinned for the common self-assign case). The assignable list is read live from Clerk per ADR 0003 (no local User table); the server resolves the chosen user's id against Clerk and snapshots their name, never trusting a client-supplied name. Ownership can be reassigned freely — if a Request already has an Owner, anyone can hand it to someone else. We accept silent reassignment because the `ActivityEvent` log records every owner change ("Bob changed owner from Alice to Carol"), which is sufficient accountability for a peer team; the trade-off is that the UI does not warn that someone else currently owns it.
